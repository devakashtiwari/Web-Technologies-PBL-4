üéØ ARIA Landmarks & Roles

ARIA landmarks help screen reader users navigate and understand page structure. They provide shortcuts to major page regions and make content organization clear.

role="banner"

Contains site-wide header, branding, and top-level navigation. Typically the page header.

role="navigation"

Groups navigation links. Use aria-label to distinguish multiple nav regions (main, secondary, social).

role="main"

Identifies the main content area. Only one per page. Helps users skip to primary content.

role="complementary"

Sidebar or supplementary content that supports the main content but can be read independently.

role="contentinfo"

Footer with copyright, links, and site-wide information. Use in <footer> elements.

role="search"

Identifies search functionality. Helps users quickly find the search feature.

üìã Semantic Lists

Unordered List Example

  • Always use semantic HTML elements (<ul>, <ol>, <li>)
  • Screen readers announce list structure and item count
  • Provides navigation shortcuts for list content
  • Never use <div> elements to create fake lists

Ordered List Example

  1. Start with semantic HTML - it's the foundation
  2. Add ARIA only when semantic HTML isn't sufficient
  3. Test with real assistive technologies
  4. Validate your code against WCAG guidelines

üì° Live Regions

Real-time Updates

aria-live attributes announce dynamic content changes:

  • aria-live="polite" - Waits for screen reader to finish before announcing (default)
  • aria-live="assertive" - Interrupts to announce immediately (use sparingly)
  • aria-live="off" - No automatic announcement

‚úÖ Accessibility Best Practices

Core Principles

  1. Use Semantic HTML First: <button>, <nav>, <main> instead of custom widgets
  2. Proper Heading Structure: Start with h1, skip levels logically (h1 ‚Üí h2 ‚Üí h3)
  3. Color Contrast: Minimum 4.5:1 for normal text, 3:1 for large text (WCAG AA)
  4. Keyboard Navigation: All functionality accessible via keyboard
  5. Focus Indicators: Visible focus states for keyboard users
  6. Alternative Text: Meaningful descriptions for images and icons
  7. Avoid Auto-play: Media shouldn't play without user action
  8. Test with Real Users: Include people with disabilities in testing
