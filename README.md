# Web-Technologies-PBL-4
Key Accessibility Features:

ARIA Landmarks - Banner, navigation, main, and contentinfo roles to help screen reader users navigate
Semantic HTML - Proper use of <nav>, <main>, <header>, <footer>, <section>, and <button> elements
Skip Link - Hidden link that appears on focus to skip to main content
Tab Component - Fully accessible tablist with keyboard navigation (arrow keys, home, end)
Forms - Properly labeled inputs with fieldsets, aria-required, and aria-describedby
Live Regions - Alert roles for dynamic content announcements
ARIA Attributes - aria-label, aria-controls, aria-hidden, aria-live, aria-selected, and aria-required
Keyboard Navigation - Full keyboard support with visible focus states
Semantic Lists - Ordered and unordered lists instead of div-based alternatives
Color Contrast - WCAG AA compliant contrast ratios throughout

Design Highlights:

Modern gradient aesthetic with professional styling
Responsive layout that works on all screen sizes
Clear visual hierarchy with semantic heading structure
Smooth interactions and transitions
Accessible form validation and feedback patterns
Interactive tab component with proper focus management
